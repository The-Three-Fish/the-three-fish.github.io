# 快速测试指南

## 🚀 立即预览更新

### 1. 启动开发服务器

```bash
cd /c/Users/Administrator/projects/the-three-fish.github.io
npm run dev
```

### 2. 访问以下页面

#### 主页（查看 oTree Launcher 服务卡片）
```
http://localhost:4321/
```

**检查要点：**
- ✅ 页面中间应该有三个服务区块
- ✅ 第一个区块标题是 "🚀 oTree Launcher - 零配置的实验研究环境"
- ✅ 内容提到"免费体验阶段"
- ✅ 有"了解详情"按钮
- ✅ 有图片轮播（3张图片自动切换）

#### 博客文章详情页
```
http://localhost:4321/blog/otree-launcher
```

**检查要点：**
- ✅ 标题："oTree Launcher：让 oTree 实验研究变得前所未有的简单"
- ✅ "如何开始使用"章节开头有"免费体验阶段"说明
- ✅ GitHub Issues 和 Discussions 链接正确
- ✅ FAQ 包含 Q7 关于收费模式
- ✅ 结尾有明确的反馈引导

## 📱 移动端测试

在浏览器中按 F12 打开开发者工具，切换到移动设备视图：

- iPhone 12 Pro (390×844)
- iPad (768×1024)
- Desktop (1920×1080)

检查布局是否正常。

## 🎨 如果图片显示不正常

当前使用的是占位图片（service-slide-1/2/3.png）。如果这些图片不存在或显示异常：

**临时解决方案：**

1. 使用单张图片代替轮播：

编辑 `src/content/homepage/index.md`，将 oTree Launcher 的 images 改为：
```yaml
images:
  - "/images/banner.png"  # 使用现有的 banner 图片
```

2. 或者暂时移除图片（仅显示文字）：

在 `src/pages/index.astro` 中临时注释掉图片部分（86-122行）。

## 🐛 常见问题

### Q: 开发服务器启动失败
```bash
# 重新安装依赖
npm install
npm run dev
```

### Q: 主页没有显示服务区块
- 检查 `src/pages/index.astro` 第 86-150 行是否取消了注释
- 确认 services 数组有内容

### Q: 博客文章 404
- 确认文件路径：`src/content/blog/otree-launcher.md`
- 检查 frontmatter 中 `draft: false`

### Q: 图片不显示
- 检查 `/public/images/` 目录下是否有对应图片
- 暂时使用现有的 banner.png 或 service-slide-*.png
